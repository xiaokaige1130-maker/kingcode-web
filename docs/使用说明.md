# KingCode 使用说明

这份文档面向日常使用，不讲架构细节，重点是怎么启动、怎么配置、怎么在 `Web` 和 `CLI` 两种模式下使用。

## 1. 项目简介

`KingCode` 是一个本地运行的代码助手，支持：

- `Web` 页面模式
- `CLI` 终端模式
- 本地工作区文件浏览和编辑
- 在工作区内运行命令
- 接入多种模型提供商

```text

```

## 5. 模型配置说明

### 5.1 当前默认模型

当前已经配置好一个可直接使用的 `DeepSeek`：

- `Profile ID`: `deepseek`
- `Type`: `openai-compatible`
- `Base URL`: `https://api.deepseek.com`
- `Path`: `/v1/chat/completions`
- `Model`: `deepseek-chat`

### 5.2 常见模型类型

- `openai-compatible`
  适合 OpenAI、DeepSeek 以及兼容 OpenAI Chat Completions 的平台
- `anthropic`
  适合 Claude 官方接口
- `gemini`
  适合 Google Gemini 官方接口
- `generic-json`
  适合自定义 JSON 请求格式的平台

### 5.3 配置项含义

- `id`
  内部标识，用于切换 profile
- `name`
  在界面里显示的名称
- `type`
  模型接口类型
- `baseUrl`
  接口域名
- `path`
  接口路径
- `model`
  模型名称
- `apiKey`
  模型密钥

## 6. Web 端使用说明

### 6.1 Web 页面主要区域

左侧区域：

- 模型配置
- 工作区路径
- 系统提示词
- 文件树

中间区域：

- 对话区
- Workflow 切换
- 提问输入框

右侧区域：

- 文件编辑器
- 命令执行区

### 6.2 常见使用流程

#### 场景一：让它分析项目

1. 启动 `Web`
2. 确认 `workspaceRoot` 正确
3. 在中间选择 `Analyze`
4. 在输入框里提问，例如：

```text
请分析这个项目的主要结构和模块关系
```

#### 场景二：让它基于某个文件回答

1. 在左侧文件树中打开文件
2. 在右侧勾选 `Include in AI context`
3. 在中间输入问题，例如：

```text
解释这个文件的作用，并指出可以优化的地方
```

#### 场景三：先跑命令，再让它分析结果

1. 在右侧 `Command Deck` 输入命令，例如：

```text
git status
```

2. 点击 `Run Command`
3. 命令输出会自动进入上下文
4. 在中间继续提问，例如：

```text
根据这个 git 状态，建议我下一步怎么处理
```

#### 场景四：直接编辑文件

1. 在文件树中打开一个文件
2. 在右侧编辑内容
3. 点击 `Save File`

### 6.3 Workflow 用途

- `Analyze`
  用于分析项目、梳理结构
- `Plan`
  用于做改动方案和拆解步骤
- `Review`
  用于代码审查、找风险
- `Implement`
  用于让模型给出具体实现建议

## 7. CLI 端使用说明

### 7.1 启动

```powershell
cd C:\Users\云电脑\Desktop\omnicode-web
npm run cli
```

启动后会看到类似提示：

```text
[KingCode CLI]

Workflow: analyze

```

### 7.2 CLI 基本规则

- 直接输入普通文本：表示向模型提问
- 输入 `/` 开头的内容：表示执行命令

### 7.3 CLI 常用命令

#### 查看帮助

```text
/help
```

#### 查看当前状态

```text
/status
```

#### 查看可用模型

```text
/profiles
```

#### 切换模型

```text
/profile deepseek
```

#### 查看可用 workflow

```text
/workflows
```

#### 切换 workflow

```text
/workflow review
```

#### 查看目录树

```text
/tree
```

或：

```text
/tree src
```

#### 列目录

```text
/ls
```

#### 打开文件

```text
/open README.md
```

#### 把文件加入上下文

```text
/include server.js
```

#### 取消文件上下文

```text
/exclude server.js
```

#### 查看已加入上下文的文件

```text
/files
```

#### 写文件

```text
/write notes.txt
```

执行后进入多行输入模式，结束时输入：

```text
.end
```

#### 运行命令

```text
/run git status
```

#### 清除上一次命令输出上下文

```text
/clearcmd
```

#### 退出 CLI

```text
/exit
```

### 7.4 CLI 使用示例

#### 示例一：分析当前项目

```text
/tree
请分析这个项目的主要模块和职责
```

#### 示例二：带文件上下文提问

```text
/include server.js
请解释这个服务端入口的工作流程
```

#### 示例三：带命令输出提问

```text
/run git status
```

## 8. Web 与 CLI 的区别

### Web 版适合

- 看文件树
- 边看边改
- 多区域同时操作
- 更直观地切换配置

### CLI 版适合

- 纯终端环境
- 快速提问
- 边跑命令边问
- 更接近命令行工作流

## 9. 常见问题

### 9.1 启动后没有响应

先检查端口是否被占用：

```powershell
netstat -ano | findstr :4780
```

### 9.2 CLI 输入后感觉“没回复”

可能原因：

- 模型接口网络慢
- API Key 不可用
- 请求失败但当前提示不明显

先切换到 `Web` 版测试同一模型是否能正常返回。

### 9.3 模型报错

重点检查：

- `baseUrl`
- `path`
- `model`
- `apiKey`
- 当前网络是否能连通对应平台

### 9.4 读不到文件

通常是因为目标文件不在 `workspaceRoot` 内。

### 9.5 别的电脑访问不了

当前项目更适合本机使用。  
如果要局域网共享访问，需要额外处理：

- 监听地址
- 防火墙
- 安全控制

因为这个工具支持文件写入和命令执行，不建议直接裸开放。

## 10. 推荐使用习惯

- 用 `Web` 版做配置和文件查看
- 用 `CLI` 版做快速问答和命令联动
- 改动前先用 `Analyze`
- 实施前切到 `Plan`
- 审查代码用 `Review`
- 需要具体写法时用 `Implement`
