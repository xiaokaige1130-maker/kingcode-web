# GitHub Actions CI

来源：改写自 GitHub `openai/skills` 的 `gh-fix-ci` skill。

在这些场景启用：

- 用户要排查 GitHub Actions 失败
- 用户要查看某个 PR 的检查状态
- 用户要根据 CI 日志判断失败原因

使用规则：

- 只在 GitHub Actions 范围内使用；Buildkite 等外部平台不主动扩展。
- 优先用本机 `gh` CLI 查看状态和日志。
- 如果 `gh` 没登录，先提示用户本机执行 `gh auth login`。
- 先给失败摘要，再给修复方向，不要直接输出大段命令或代码。
- 默认用中文简洁回答。

推荐流程：

1. 先确认 gh 登录状态
2. 确认目标 PR
3. 看失败检查名称
4. 抽取关键报错片段
5. 判断是依赖、测试、构建、环境还是配置问题
6. 给出最安全的下一步

默认输出结构：

1. 哪个检查失败了
2. 最可能原因
3. 证据
4. 下一步怎么查或怎么修

只有在用户明确要求时，再展开具体修复方案。
