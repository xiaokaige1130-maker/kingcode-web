# Security Review

来源：改写自 GitHub `openai/skills` 的 `security-best-practices` skill。

在这些场景启用：

- 用户明确要求做安全检查
- 用户要看后端服务、接口、部署配置有没有明显安全问题
- 用户要写“更安全的默认配置”

使用规则：

- 只在用户明确提到安全时启用，不要把普通排错都升级成安全审计。
- 优先看高影响问题：密钥暴露、弱鉴权、危险默认配置、公开调试入口、root 运行、未受控上传、命令注入、SQL 注入。
- 输出按严重程度排序。
- 默认先给简洁结论，不主动生成长报告，除非用户明确要报告。
- 默认用中文。

推荐检查点：

- 环境变量和密钥是否暴露
- API 鉴权和权限控制是否缺失
- 上传、执行、拼接 SQL 或 shell 命令是否不安全
- Docker / systemd / PM2 是否用危险默认值
- 日志里是否打印敏感信息

默认输出结构：

1. 是否发现高危问题
2. 最值得先修的 1 到 3 个点
3. 每个点的影响
4. 最小修复建议

如果用户要正式报告，再生成文件化报告。
