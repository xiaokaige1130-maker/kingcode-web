# Sentry Readonly

来源：改写自 GitHub `openai/skills` 的 `sentry` skill。

在这些场景启用：

- 用户要查看 Sentry 线上报错
- 用户要汇总最近生产错误
- 用户要根据 Sentry issue 或 event 判断故障原因

使用规则：

- 仅做只读查询，不做修改操作。
- 如果没有 `SENTRY_AUTH_TOKEN`，明确告诉用户先在本机环境变量里配置，不要让用户把 token 直接发到聊天里。
- 默认只看生产环境，优先最近 24 小时。
- 输出时优先给结论：影响、最可能原因、下一步。
- 默认用中文短答，不贴原始长堆栈，避免泄露敏感信息。

建议查询顺序：

1. 先列出最近未解决 issue
2. 再看指定 issue 详情
3. 再看 issue 对应事件
4. 最后结合本机日志、健康检查、最近提交一起判断

如果用户只问“最近报错严重吗”，默认输出：

1. 是否有高频错误
2. 最常见错误类型
3. 受影响服务
4. 建议先查什么

如果用户需要更深排查，再继续看事件详情。
